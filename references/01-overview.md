## Purpose

Use this reference as the entry point for n8n custom node work. It defines the lifecycle and links to planning, building, testing, deployment, and reference material.

## Lifecycle Summary

1. Plan your node (type, style, UX, structure).
2. Build using declarative or programmatic style.
3. Test with local execution and linter checks.
4. Deploy as community package or private node.
5. Validate against file structure, credentials, codex, and UI references.

## Prerequisites

- JavaScript/TypeScript familiarity.
- Ability to manage local development environment and git workflows.
- npm packaging knowledge (including publishing).
- n8n data and item-linking basics.
