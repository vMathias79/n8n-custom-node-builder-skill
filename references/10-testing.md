## Required Testing Modes

Use both:

1. Manual local execution in n8n.
2. Automated/static lint checks.

## Local Run Workflow

1. Install n8n globally:
   ```shell
   npm install n8n -g
   ```
2. Build and link your node package from its root:
   ```shell
   npm run build
   npm link
   ```
3. Link package inside the n8n custom extensions directory (`~/.n8n/custom/` by default, or the path set by `N8N_CUSTOM_EXTENSIONS`):
   ```shell
   npm link <node-package-name>
   ```
   If `~/.n8n/custom/` doesn't exist, create it and run `npm init` inside it.
4. Start n8n:
   ```shell
   n8n start
   ```
5. Search for your node in the node panel by **node name** (not the npm package name).

## Lint Workflow

Linter configuration is provided by `@n8n/node-cli` via `eslint.config.mjs` (do not edit this file). In a community package the linter runs automatically after `npm install` and before publishing. You can also run it manually:

```shell
npm run lint       # report issues
npm run lint:fix   # auto-fix where possible
```

VS Code lints in the background when the ESLint extension is installed. Use Quick Fix → "Disable {rule} for this line" sparingly if a rule is incorrectly flagged.

## Troubleshooting Highlights

- Unknown credentials type: confirm credential class/property names and references match.
- Node not visible in Add Node menu: verify node registration under the `n8n` key of `package.json`.
- Icon doesn't show: icon file must live beside the node, be `.png` or `.svg`, and be referenced with the `file:` prefix (e.g. `file:friendGrid.svg`).
- Icon doesn't fit: SVG canvas must be square; PNG must be 60x60.
- Description/UI changes don't appear on refresh: stop n8n (`ctrl+c`), restart it, and re-run `npm link` if needed.
- Linter keeps flagging file-name case on Windows: known Windows issue with case-only renames.
